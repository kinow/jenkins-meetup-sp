O que plug-ins podem fazer?

  • Modificar a UI
  • Integrar Jenkins com outros sistemas
  • Executar um parser
  • Coletar métricas do build
  • Chamar outras API's como Twitter, JMS, SAP, etc
  • Utilizar serviços em nuvem
  • Rodar Jenkins como nó Hadoop
  • Chamar um cluster PBS e executar um programa MPI (BioUno)
  • Há diversos extension points

Exemplos de extension points

  • hudson.slaves.ComputerListener (hadoop, keep-slaves-offline, monitoring)
  • hudson.slaves.Cloud (Amazon EC2, Lab Manager)
  • hudson.tasks.test.TestResultParser (utpsql, core)
  • hudson.tasks.Builder (TestLink, Ant, CCM, Gradle, MsBuild)

Preparando o ambiente

  • Java 5 6+
  • Eclipse (IntelliJ, NetBeans, Vim, ...)
  • Maven 3.x
  • Modificar settings.xml do Maven
  • Conexão boa com a Internet (yup, o Maven vai baixar a Internet :)

Criando o plug-in hello world

  • mvn hpi:create
  • Em caso de erros, use -X -e
  • Depois da primeira vez pode user o -o
  • mvn package para empacotar seu plug-in
  • mvn clean hpi:run para executar
  • E então http://localhost:8080

Entendendo a estrutura dos plug-ins

  • src/main/java - Classes do seu plug-in
  • src/main/resources - jelly, groovy, html, ...
  • src/main/webapp - css, js, images, html, ...

Vamos modificar o nosso plug-in

  • Vamos usar outro extension point
  • Interagir com um sistema externo
  • Usar uma API para acessar este sistema
  • Criar uma action
  • Enfrentar os problemas que aparecerem
  • Copiar outros plug-ins (!)
